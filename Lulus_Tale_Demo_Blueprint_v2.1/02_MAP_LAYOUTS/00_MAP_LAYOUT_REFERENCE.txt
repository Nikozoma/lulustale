All maps use 32 game-world pixels per tile and four semantic layers:
- ground
- structures
- objects
- markers

The visual companions use the approved current full-map reconstructions under
public/assets/maps/. Semantic JSON remains authoritative for collision,
walkability, interactions, transitions, and spawns.

The runtime selects the night companions only when the manual phase state is
night. Both phases intentionally share the same semantic maps so collision,
markers, transitions, and quest interactions cannot drift apart.

Overworld.visual.json contains six active placements:
- one current walk-behind base,
- four upper-building foreground sections,
- one transparent tree-canopy foreground.

The building foreground sections are opaque normally and fade only when Lulu's
feet enter their four-row enlarged walk-behind zones. The tree canopy remains last in draw
order. See MAP_READINESS_REPORT.txt and
MAP_CHANGE_MANIFEST_building_foreground_pass.txt for the current contract.

Overworld.night.visual.json mirrors the same six-placement structure using the
supplied night walk-behind base, four night building sections, and night tree
canopy. Dedicated night visuals replace the old blanket-tint presentation.

The Overworld day/night visuals are 3072 x 2176 pixels and retain normal 32 x
32 game tiles. Each cell of the approved 48 x 34 semantic source became an
exact 2 x 2 block, while point markers remain singular at tuned positions.
