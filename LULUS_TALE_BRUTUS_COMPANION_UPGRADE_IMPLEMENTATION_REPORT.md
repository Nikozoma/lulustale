# Lulu's Tale — Brutus Companion Upgrade Implementation Report

## Summary

Implemented the requested Brutus companion upgrade on top of `Lulus_Tale_Day1_Demo_Main_Menu_v1`.

Completed:

- Brutus interaction submenu integrated into the existing top-right world Actions menu.
- Pet, Feed, Sit / Stay, Follow, Lie Down, and Fetch are available when Lulu is near Brutus during Day Mode.
- Brutus remains a separate runtime entity.
- Existing petting, feeding, synchronized command metadata, root anchors, facings, and timing remain in use.
- Sit / Stay and Lie Down use the supplied command/transition animation assets.
- Follow resumes the existing recent-player-path companion following system.
- Fetch uses the supplied `Lulu_throw_toy`, `Brutus_play_ready`, and `Brutus_carry_pose` assets plus the authoritative fetch interaction metadata.
- The fetch prop uses the existing real bone sprite from `TopDownHouse_SmallItems.png` as a separate runtime prop; it is not baked into Lulu or Brutus.
- Fetch includes throw, chase, pickup/carry, return, and happy completion states.
- Fetch checks for sufficient open collision-safe space before starting.
- Brutus continues to follow safely across Home, Overworld, and Charles Jr. using the existing transition placement/recovery logic.
- Home ambient behavior was expanded: while Lulu remains idle and Brutus is in Follow mode, Brutus can wander among safe indoor rest locations, sniff, sit, or lie down. Moving Lulu immediately cancels ambient wandering and returns Brutus to following behavior.
- Stay/Sit or Lie Down commands suppress ambient roaming until Follow is selected.
- Companion command pose is included in autosave/backup save snapshots as an optional backwards-compatible field.
- Existing standalone Pet/Feed/Stay UI buttons are hidden so companion actions are presented through the requested interaction menu; the Run control remains available.

Nighttime Brutus behavior remains unchanged and intentionally deferred: Brutus is not rendered/updated in Night Mode.
