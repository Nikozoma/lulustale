IMPLEMENTATION APPROACH
-----------------------

The night maps were implemented as visual variants of the three existing maps,
not as duplicate gameplay worlds.

Daytime companions:
- Home.visual.json
- Charles.visual.json
- Overworld.visual.json

New nighttime companions:
- Home.night.visual.json
- Charles.night.visual.json
- Overworld.night.visual.json

Runtime selection:
- quest.isNight false -> daytime companion
- quest.isNight true -> nighttime companion

The same semantic files are used in both phases:
- Home.json
- Charles.json
- Overworld.json

This is intentional. Collision, doors, markers, interactions, NPC spawn
anchors, transitions, and quest behavior therefore remain identical between
day and night unless a later gameplay system explicitly adds a phase condition.
Visual artwork never creates or changes collision.

FALLBACK BEHAVIOR
-----------------

The visual selection helper safely falls back to a map's daytime visual if a
future map does not have a night companion. Only that fallback case receives
the old generic blue night tint.

All three current maps have dedicated night companions, so their supplied night
art is not darkened a second time.

1. One full-map night walk-behind base on the objects layer.
2. North apartment upper foreground, order 80.
3. Charles Jr. upper foreground, order 81.
4. Lulu's apartment upper foreground, order 82.
5. South apartment upper foreground, order 83.

Building behavior preserved:

Tree behavior preserved:
- 21 semantic tree anchors.
- Night canopy drawn after Lulu/entities.
- Day and night tree alpha masks are byte-for-byte identical.

Alpha validation:
- Night building foreground nontransparent pixels: 98,304.
- Night tree foreground nontransparent pixels: 43,920.
- Day/night building alpha masks: exact match.
- Day/night tree alpha masks: exact match.

WHY THE FOREGROUND-PACK OVERWORLD BASE IS ACTIVE
------------------------------------------------

Night.OverW.zip contains the approved normalized night reference.
Night.OW.ForG.zip contains the dedicated runtime layers fitted to the active
daytime map coordinates and existing foreground masks.

The foreground pack's walk-behind base is used at runtime because it preserves
exact alignment with:

- the current Overworld semantic grid,
- the four building occlusion rectangles,
- the 96 walkable building tiles,
- the night building foreground,
- the night tree-canopy mask.

The approved normalized Night.OverW image is still preserved as the visual
reference for future refinement. Nothing from either source pack was discarded.
