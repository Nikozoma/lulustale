LULU'S TALE — OVERWORLD 96x68 IMPLEMENTATION REPORT
====================================================

Date: 2026-07-12
Status: IMPLEMENTED; AUTOMATED VALIDATION AND PRODUCTION BUILD PASS


SUMMARY
-------

The approved Overworld was enlarged from 48 x 34 to 96 x 68 normal game
tiles. The engine tile size remains 32 x 32 pixels. No camera zoom, player
scale, movement scale, per-map scale, or engine tile-size workaround was used.

Day and night base art, building foregrounds, tree foregrounds, visual
companions, semantic layers, markers, transitions, manifests, documentation,
tests, and source tile packs now agree on 96 x 68 tiles / 3072 x 2176 pixels.

The visual enlargement is exact nearest-neighbor 2x sampling. Every original
pixel becomes a 2 x 2 pixel block, preserving the approved composition,
landmarks, proportions, style, and day/night alignment.


SEMANTIC AND MARKER RESULT
--------------------------

Ground, structures, and objects were expanded so every original semantic cell
became an exact 2 x 2 block. This preserves roads, sidewalks, parking, grass,
building footprints, bushes, hedges, fences, trees, and event-use areas.

Point markers remain singular rather than becoming four duplicate triggers.
They use the lower-right 32 x 32 cell of the corresponding enlarged 2 x 2
block. Final Overworld marker coordinates are:

- player_door: 67,37
- transition_to_home: 67,39
- player_spawn: 67,41
- charles_jr_door: 29,39
- transition_to_charles_jr: 29,41
- npc_spawn: 23,49 and 85,49

All markers are in bounds and have no structure/object collision. Home and
Charles Jr. return behavior continues to use these existing marker IDs, so no
interior-map compatibility edit was required.

KNOWN ISSUES / RISKS
--------------------

- No automated data, unit-test, or build issue remains.
- The exact 2x art enlargement intentionally produces larger pixel blocks; it
  is not a redraw and therefore preserves the approved composition exactly.
- A marker cannot sit at the exact doubled center of an old tile because that
  point lies on a new tile boundary. Singular markers use the lower-right
  candidate, a 16-pixel offset from that boundary, and are verified open.
- The game currently preloads day and night visual companions. Enlarged full-
  map PNGs increase decoded image memory on mobile even though compressed PNG
  sizes are moderate. This existing loader behavior was not rewritten because
  the request prohibited unrelated engine changes. Test on the minimum target
  phone; lazy phase/map loading is the logical optimization if memory pressure
  appears.
